# 11. 製作作品集網站

## 作品集網站的意義

作品集網站是創作者的「視覺履歷」。

它不是把所有作品一股腦兒放上去，而是：

> **有策略地展示你最好的作品，讓目標觀眾相信你的能力**

### 作品集網站的目的

| 目的 | 對象 |
|------|------|
| 求職應徵 | HR、設計主管 |
| 接案曝光 | 潛在客戶 |
| 品牌建立 | 業界同儕 |
| 自我記錄 | 自己 |

## 作品集網站的必備頁面

### 必備頁面

```
1. 首頁 (Home)
   └→ 立即了解你是誰、做什麼

2. 作品集 (Portfolio/Work)
   └→ 展示你的最佳作品

3. 關於 (About)
   └→ 你是誰、為什麼找你

4. 聯絡 (Contact)
   └→ 如何找到你
```

### 可選頁面

- **服務 (Services)**：如果你提供付費服務
- **流程 (Process)**：說明合作方式
- **Blog/教學**：分享知識，建立專業形象

## 選擇平台

### 作品集平台比較

| 平台 | 優點 | 缺點 | 適合 |
|------|------|------|------|
| Behance | 流量大、業界認可 | 自訂有限 | 設計師標配 |
| Dribbble | 設計師社群活躍 | 需要申請 | UI/UX 設計師 |
| Cargo | 設計感強 | 功能簡單 | 藝術家、攝影師 |
| Squarespace | 專業模板多 | 費用較高 | 所有人 |
| Wix | AI 建站 | 設計感一般 | 初學者 |
| 自架網站 | 完全自由 | 需要維護 | 進階用戶 |

### 本章推薦方案

- **首選**：Squarespace（設計質感佳、操作簡單）
- **備選**：Wix（功能完整）
- **搭配**：Behance 作為作品展示補充

## Squarespace 實作

### 步驟 1：選擇模板

1. 前往 `squarespace.com`
2. 點擊「Get Started」
3. 瀏覽「Portfolio」或「Design」類別模板
4. 選擇一個你喜歡的風格

**推薦模板**：
- **Avenue**：攝影師適用，大圖展示
- ** Tudor**：設計師適用，網格式作品牆
- **Paloma**：藝術家適用，藝廊風格

### 步驟 2：設定基本資訊

進入編輯器後，設定：

1. **Site Title**：你的名字或工作室名
2. **Tagline**：一句話介紹
3. **Logo**：上傳你的 Logo 或文字

### 步驟 3：上傳作品

1. 點擊「Pages」→「Add」
2. 選擇「Gallery」或「Portfolio」
3. 上傳圖片或建立專案

**專案建立技巧**：
```
每個專案包含：
- 封面圖（最重要的視覺）
- 專案名稱
- 客戶/年份（可選）
- 專案說明
- 詳細圖片
```

### 步驟 4：編輯作品說明

好的作品說明讓作品更有說服力：

```
格式：
[專案名稱] - [客戶/品牌]
[一句話描述專案目標]
[你的角色]：負責什麼
[使用工具/技術]：[工具名稱]
[成果/impact]：[如果有數據]
```

**範例**：
```
星巴克品牌形象更新 - 星巴克台灣
重新設計品牌視覺系統，提升年輕族群好感度。
我的角色：品牌策略、視覺設計
使用工具：Illustrator, Photoshop, Figma
成果：品牌好感度提升 25%
```

## AI 幫你撰寫作品集內容

### About 頁面 Prompt

```
角色：你是一個有創意的個人品牌教練

任務：幫我撰寫「關於」頁面的文案

背景：
- 名字：[名字]
- 職業：[設計師/攝影師/藝術家]
- 專長：[具體擅長的領域]
- 經歷：[重要經歷簡述]
- 風格：[你的設計風格或理念]

需求：
1. 總長度：300-400 字
2. 結構：
   - 開頭：我是誰，專注什麼
   - 中間：背景故事、設計理念
   - 結尾：為什麼找你、合作方式
3. 語氣：自信但不傲慢，有個性
4. 加入一個能讓人記住你的小故事或特色
```

### 作品說明 Prompt

```
角色：你是一個專注於設計的文案專家

任務：幫我撰寫作品的文案

作品：
- 名稱：[作品名稱]
- 類型：[海報/網站/品牌形象等]
- 客戶：[客戶名稱，如果有]
- 目標：[這個專案的目標]
- 我的角色：[負責什麼]
- 使用工具：[工具]

需求：
1. 每個作品 50-100 字
2. 重點說明：
   - 專案背景/挑戰
   - 你的解決方案
   - 最終成果（如果有）
3. 語氣專業、有說服力
4. 避免空泛形容詞，用具體描述
```

### 合作邀約文案 Prompt

```
角色：你是一個善於溝通的創意工作者

任務：幫我撰寫聯絡區或合作邀約的文案

背景：
- 提供的服務：[類型]
- 適合找我的人：[理想客戶畫像]
- 合作方式：[專案/駐點/顧問等]
- 聯絡方式：[email/表單連結]

需求：
1. 50-100 字
2. 語氣熱情但專業
3. 清楚說明：
   - 適合找我解決什麼問題
   - 如何開始第一步
4. CTA 明確
```

## 作品集視覺設計原則

### 圖片展示技巧

1. **高品質優先**：寧願少放也不要放低品質圖片
2. **統一尺寸**：同一類作品用相同的展示比例
3. **留白空間**：讓作品有呼吸的空間
4. **細節放大**：重要的設計細節可以單獨展示

### 排版原則

```
設計師作品集排版參考：

█████████████
█████████████  ← 大圖展示（70% 空間）
█████████████

████████████
████████████  ← 中圖展示（40% 空間）
     或
████████████  ← 左圖右文（50%/50%）
████████████
▓▓▓▓▓▓▓▓▓▓▓▓
▓▓▓▓▓▓▓▓▓▓▓▓  ← 細節展示（小圖）

████████████
████████████
████████████  ← 網格展示（多圖）
████████████
```

### 配色策略

作品集網站的配色建議：

1. **極簡黑白**：讓作品本身成為視覺焦點
2. **中性灰白**：乾淨的背景，突出作品
3. **品牌色系**：如果有明顯的個人品牌色彩

**避免**：
- 太花俏的背景
- 與作品競爭注意力的設計
- 過多裝飾元素

## 各類創作者作品集要點

### 平面設計師

```
必備元素：
✓ Logo 設計作品
✓ 品牌視覺系統
✓ 包裝設計
✓ 海報/印刷品

展示重點：
- 設計過程（草稿→完稿）
- 設計說明（為什麼這樣設計）
- Before/After
```

### 網頁/UI 設計師

```
必備元素：
✓ 網站設計 mockup
✓ App 介面設計
✓ 使用者流程圖
✓ 響應式設計展示

展示重點：
- 完整的頁面截圖
- 動態效果演示（GIF/影片）
- 設計系統元件
```

### 攝影師

```
必備元素：
✓ 系列作品（不是單張散落）
✓ 拍攝主題明確
✓ 多元風格展示（可選）

展示重點：
- 高品質大圖
- 同系列作品集結
- EXIF 或拍攝資訊（可選）
```

### 插畫師

```
必備元素：
✓ 插畫作品集
✓ 風格展示（多種風格可加分）
✓ 商業應用案例（可選）

展示重點：
- 完整插畫作品
- 角色設計展示
- 創作過程
```

### 影像/動態設計師

```
必備元素：
✓ 作品影片（可嵌入 YouTube/Vimeo）
✓ 靜態 frame 截圖
✓ 設計過程

展示重點：
- 影片 demo reel
- 可播放的 GIF 或短片
- 幕後製作花絮
```

## AI 幫你生成作品集素材

### 概念圖生成 Prompt

```
用 Midjourney：

「[描述你需要的概念圖],
digital illustration, [風格描述],
professional design portfolio quality,
high detail, 4k」

範例：
「Abstract geometric shapes forming a modern logo concept,
minimalist digital illustration, clean lines,
professional design portfolio quality, high detail, 4k」
```

### Mockup 生成 Prompt

```
用 AI 或線上工具：

在 Smartmockups 或 Placeit 上傳你的設計，
選擇合适的場景生成 mockup。

常用場景：
- 手機展示（App 介面）
- 網站展示（電腦螢幕）
- 包裝展示（產品櫃）
- 名片展示（辦公桌面）
```

## 作品集檢查清單

### 內容檢查

- [ ] 至少有 6-10 個代表作品
- [ ] 作品說明清楚、有深度
- [ ] About 頁面讓人記住你是誰
- [ ] 聯絡方式容易找到
- [ ] 有明確的 CTA（諮詢/合作）

### 設計檢查

- [ ] 作品圖片品質高（至少 1500px 寬）
- [ ] 排版整齊、留白適當
- [ ] 網站風格與作品調性一致
- [ ] 手機版顯示正常
- [ ] 載入速度合理

### 技術檢查

- [ ] 所有連結正常
- [ ] 圖片有替代文字
- [ ] 有 SSL 憑證
- [ ] 社群連結有效

### 策略檢查

- [ ] 作品按時間或類型有序排列
- [ ] 最好的作品放在最前面
- [ ] 有針對目標觀眾展示重點
- [ ] 更新頻率合理（定期更新）

## 持續經營作品集

### 定期更新

- 每完成一個專案就更新
- 至少每季檢視一次
- 移除過時或代表性不足的作品

### 收集回饋

- 問客戶能否使用在他們網站上
- 收集客戶的見證或回饋
- 記錄專案的數據成果

### 建立外部連結

- 連結到 Behance/LinkedIn
- 在社群媒體分享作品
- 參與設計社群增加曝光
