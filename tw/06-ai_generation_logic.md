# 6. AI 生成網站的邏輯與限制

## AI 是如何「思考」的

要有效使用 AI 工具，你需要了解它的工作方式。不是因為要成為專家，而是因為知道「為什麼」能幫助你更好地「怎麼做」。

### AI 不是在「設計」，而是在「匹配」

當你向 AI 描述需求時，AI 會：

1. **解析你的描述**：把自然語言轉換成關鍵特徵
2. **搜索資料庫**：找到符合這些特徵的模板或模式
3. **組合生成**：把找到的元素組合成一個網站

```
你的描述 ──→ AI 分析 ──→ 資料庫匹配 ──→ 組合生成 ──→ 輸出網站

例如：
「簡約風格」──→ 分析：白底、簡單排版、大量留白 ──→
搜尋相關模板 ──→ 組合各區塊 ──→ 輸出結果
```

### AI 善於「常見模式」，不擅長「獨特創意」

AI 的訓練資料是現有的網站和設計。這意味著：

| AI 擅長 | AI 不擅長 |
|---------|----------|
| 常見的網站類型 | 完全前所未見的創意 |
| 標準的排版模式 | 打破常規的布局 |
| 流行的設計趨勢 | 引領趨勢的創新 |
| 明確的功能需求 | 模糊的感覺描述 |

## AI 生成的基本邏輯

### 邏輯 1：從模板出發

大多數 AI 建構工具的底層是大量預先設計好的模板。AI 的工作不是從零創造，而是：

- 理解你要的是哪種類型的網站
- 選擇最適合的模板
- 根據你的描述微調細節

### 邏輯 2：區塊化組裝

現代網站是由「區塊」(Section/Block) 組成的：

```
┌──────────┐
│  Header  │  ← 導航區塊
├──────────┤
│  Hero    │  ← 主視覺區塊
├──────────┤
│ Features │  ← 特色區塊
├──────────┤
│  Footer  │  ← 頁尾區塊
└──────────┘
```

AI 會選擇適合的區塊，然後按照邏輯順序組裝。

### 邏輯 3：遵循設計原則

AI 生成的網站通常遵循基本設計原則：

- **層次分明**：標題、正文、說明的字體大小有區別
- **對齊整齊**：元素會對齊邊緣或中心
- **色彩和諧**：選擇相配的顏色組合
- **留白適當**：避免過度擁擠

## AI 的強項

### 強項 1：速度

AI 能在幾秒到幾分鐘內生成一個網站，這是人工無法比擬的。

### 強項 2：一致性

AI 會保持整個網站的風格一致，不會出現這頁用藍色、那頁用紅色的問題。

### 強項 3：完整性

AI 傾向於包含所有必要的元素，不容易遺漏聯絡資訊、隱私政策等。

### 強項 4：響應式設計

幾乎所有 AI 工具都會自動生成「在手機上也能正常顯示」的版本。

### 強項 5：SEO 基礎

很多 AI 工具會自動幫你設定基本的 SEO（搜尋引擎優化）標籤。

## AI 的限制

### 限制 1：版權和獨特性

**問題**：AI 生成的網站可能與其他用戶的網站相似。

**原因**：畢竟大家都是從同一個模板庫選擇的。

**解決方法**：
- 深入客製化調整
- 更換為自己拍攝的圖片
- 添加獨特的內容和文案

### 限制 2：複雜功能

**問題**：太複雜的功能 AI 無法完美生成。

**原因**：複雜功能需要精確的邏輯設計，AI 在這方面較弱。

**AI 難以勝任**：
- 複雜的會員系統
- 即時更新的動態資料
- 需要第三方 API 串接的功能
- 客製化的結帳流程

### 限制 3：特殊設計需求

**問題**：如果你有非常具體的設計要求，AI 可能難以滿足。

**困難的例子**：
- 「我要一個會員登入後首頁內容不同的網站」
- 「我要一個可以即時翻譯成 20 種語言的網站」
- 「我要一個網站視覺效果完全配合我的品牌影片」

### 限制 4：理解模糊需求

**問題**：AI 處理模糊描述的能力有限。

**例子**：
- ❌ 「要有那種感覺」
- ❌ 「差不多就像那個網站」
- ❌ 「別人看了會覺得很酷」

這些 AI 完全無法理解，需要轉換成具體描述。

### 限制 5：最新技術和趨勢

**問題**：AI 的訓練資料有截止日期，可能不包含最新的設計趨勢。

**例子**：
- 最新流行的滾動效果
- 新興的互動設計模式
- 最近流行的配色

## 最佳化你的期望

### 現實的期待

| 你想要的 | AI 能做到的 | 需要額外努力的 |
|---------|------------|---------------|
| 快速有個網站 | ✓ | - |
| 基本的專業外觀 | ✓ | - |
| 手機適配 | ✓ | - |
| 基本 SEO | ✓ | - |
| 獨特的視覺效果 | 部分 | 需要客製化 |
| 複雜功能 | 有限 | 可能需要專業開發 |
| 完全客製化 | 困難 | 學習程式或找設計師 |

### 不要期待的

- 完全符合你腦中想像的網站一步到位
- 生成的文案不需要修改就能用
- 生成的圖片完全符合品牌形象
- 網站上線後不需要任何調整

### 可以期待的

- 一個「60 分」的網站作為起點
- 大幅節省從零開始的時間
- 有趣的設計靈感
- 節省大部分的設計工作

## 如何彌補 AI 的限制

### 面對「相似度」問題

```
1. 深入客製化
   └→ 調整顏色到品牌色
   └→ 替換所有示範圖片
   └→ 修改文案為原創內容

2. 添加個人元素
   └→ 加入你的照片
   └→ 講述你的故事
   └→ 展示真實的案例
```

### 面對「複雜功能」問題

```
1. 評估功能是否必要
   └→ 真的需要複雜功能嗎？
   └→ 有沒有更簡單的替代方案？

2. 分階段實現
   └→ 第一版先上基本功能
   └→ 後期再加入複雜功能

3. 使用第三方工具
   └→ 會員系統：用 MemberPress
   └→ 預約系統：用 Calendly
   └→ 論壇：用 Discourse
```

### 面對「模糊需求」問題

```
1. 練習將感覺轉化為具體描述
   └→ 「很酷」→「讓人想點擊的 CTA 按鈕」

2. 提供具體範例
   └→ 找到你喜歡的網站，說「像這個」

3. 使用問答方式
   └→ 讓 AI 問你問題，你回答
```

## 實用心態

記住這個心態：

> **AI 是你的超級助理，不是萬能魔法師。**
> 
> 它能幫你處理 80% 的基礎工作，讓你專注在真正重要的 20%。
> 
> 那 20% 就是：你的獨特創意、品牌故事、個人風格。
