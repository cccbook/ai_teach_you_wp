# 5. 用對話描述你想要的網站

## 為什麼描述很重要

AI 生成網站的品質，高度取決於你如何描述你的需求。一個模糊的描述會產生一個模糊的網站；一個清晰的描述則會產生一個接近你想像的網站。

```
模糊描述                    精準描述
─────────────────────────────────────────────────
「我要做一個網站」          「我要做一個個人攝影師的
                            黑白風格作品集網站」

「要好看的」                「要簡約、北歐風格，
                            以白色和淺灰色為主」

「要有特色」                「要在首頁有一個大視覺圖，
                            下面三個服務項目的卡片」
```

## 描述的基本框架：5W1H

當你要向 AI 描述網站需求時，可以用這個框架：

| 要素 | 問題 | 範例回答 |
|------|------|---------|
| **What** | 做什麼網站？ | 個人部落格、作品集、电商店 |
| **Who** | 給誰看？ | 潜在客戶、雇主、粉丝 |
| **Why** | 為什麼需要？ | 展示作品、銷售產品、分享知識 |
| **Where** | 在哪裡使用？ | 台灣市場、全球華人、科技產業 |
| **When** | 什麼時候需要？ | 立即上線、一个月后 |
| **How** | 想要怎樣呈現？ | 簡約、華麗、專業、親切 |

## 描述的層次結構

### 第一層：網站類型

先確定網站的類型：

- **個人部落格**：分享文章、想法、知識
- **品牌形象網站**：介紹公司或個人品牌
- **作品集網站**：展示創意作品或服務
- **電商網站**：販售商品
- **登陸頁面**：單一頁面推廣特定產品或活動

### 第二層：核心功能

確定網站需要哪些功能：

- 部落格系統
- 作品展示（圖庫）
- 聯絡表單
- 線上預約
- 會員系統
- 購物車
- 論壇/留言板

### 第三層：視覺風格

描述你想要的視覺風格：

**顏色偏好**：
- 「以黑白為主」
- 「企業藍和白色」
- 「大地色系」
- 「繽紛多彩」

**風格描述**：
- 「極簡主義」
- 「現代科技感」
- 「溫暖亲切」
- 「優雅奢華」

**排版偏好**：
- 「大量留白」
- 「資訊密集」
- 「卡片式布局」

## 實際對話範例

### 範例 1：餐廳網站

**錯誤示範**：
```
做一個餐廳網站
```

**正確示範**：
```
我要做一個義大利餐廳的網站，
目標客群是 25-40 歲的年輕上班族，
風格要有質感但不死板，
用暖色調（米色、棕色、酒紅色），
首頁要有大大的菜單展示區，
要有一個可以看照片的 gallery，
還有線上訂位的功能，
最後要有地址和地圖。
```

### 範例 2：自由接案設計師

**錯誤示範**：
```
做一個設計師的 portfolio
```

**正確示範**：
```
我是 UI/UX 設計師，想要一個作品集網站，
主要給潛在客戶和 HR 看的，
想要乾淨俐落的現代風格，
黑底白字為主，輔以一點亮色作為點綴，
首頁放一個簡短的自我介绍和 CTA 按鈕，
作品集頁面要用網格展示，每個作品可以點進去看详情，
要有聯絡表單功能。
```

### 範例 3：電商登陸頁

**錯誤示範**：
```
做一個賣東西的頁面
```

**正確示範**：
```
我要做一個護膚品的單一產品登陸頁，
目標是促進購買，
要有產品特色介紹、使用方式說明、
顧客評價、FAQ、常見問題區，
最後要有購買按鈕和限時優惠倒數，
整體風格乾淨清爽，用白色和淺綠色。
```

## 向 AI 描述的技巧

### 技巧 1：先說結論

把最重要的資訊放在最前面：

```
好的說法：「我要一個極簡風的攝影師作品集，重點是
展示作品，所以要讓照片最大化，文字要少，導航要
隱藏，點進去才看到完整內容。」

不好的說法：「我先跟你說，我是一個攝影師，我有很
多作品，我想要...（中間省略500字）...所以我需要
一個網站來展示。」
```

### 技巧 2：提供參考

如果有喜歡的網站，直接提供連結：

```
「我要做一個像 Apple 官網那樣乾淨的科技產品網站，
可以滑動的大圖片，文字很少但很有力量。」
```

### 技巧 3：列舉必要元素

用清單的方式確保重要元素不被遺漏：

```
網站需要包含：
1. 首頁大視覺 + 公司標語
2. 服務項目（3-4 項）
3. 團隊成員介紹
4. 客户 Logo 展示
5. 聯絡表單
6. 地圖
```

### 技巧 4：說明不要什麼

有時候說明「不要什麼」也很重要：

```
「我要一個瑜伽教室的網站，
要溫暖自然的感覺，
不要太多紫色（那是競爭對手的顏色），
不要 flash 動畫，要用手機也能看。」
```

### 技巧 5：分層次描述

先給大方向，再補充細節：

```
第一輪：「我要做一個網紅的電商網站」
↓
第二輪：「衣服和飾品為主，風格要時尚、年輕」
↓
第三輪：「具體來說，要有新品區、暢銷區、直播專區，
每個商品頁要有尺寸表、搭配推薦」
```

## 使用 AI 協助你描述

你也可以先問 AI 幫你整理思路：

### 向 ChatGPT 提問

```
「我要做一個 XXX 類型的網站，
但我不知道怎麼描述需求，
可以問我一些問題幫我整理嗎？」
```

AI 會引導你回答一系列問題，幫你系統性地思考需求。

### 讓 AI 幫你整理

把你模糊的想法丟給 AI：

```
「我有一個大概的想法：我想做一個幫助新手
父母學習育兒知識的網站，但還不太清楚
具體要怎麼做。你可以幫我整理一下可能的
功能和架構嗎？」
```

## 描述後的迭代過程

AI 生成後，如果結果不如預期怎麼辦？

### 步驟 1：指出問題

不要只說「不好看」，要具體指出：

```
「這次的問題是：
1. 顏色太暗了，要更明亮
2. 字體太小，老人家看不清楚
3. 想要logo在左邊，不要置中」
```

### 步驟 2：一次不要改太多

每次修改 2-3 個地方，改太多 AI 可能會遺漏或誤解。

### 步驟 3：給予肯定

如果 AI 做了對的東西，先說謝謝再提修改：

```
「很不錯！首頁的大圖效果很好。
小調整：1. 把背景色從深灰改成白色 2. 
把聯絡電話移到頁尾」
```

## 常見描述句型

記住這些常用句型，描述更順暢：

### 功能類
- 「需要有...功能」
- 「必須有...才能運作」
- 「可以沒有...但是要有起碼的...」

### 風格類
- 「風格要像...」
- 「不要...的感覺」
- 「整體氛圍是...」

### 布局類
- 「...要在上面/下面/左邊/右邊」
- 「...要大一點/小一點」
- 「...中間放...，旁邊放...」

### 優先順序類
- 「最重要的是...」
- 「...可以沒有但...一定要有」
- 「如果只能選一個功能，要...」
