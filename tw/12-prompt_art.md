# 12. 精準下 Prompt 的藝術

## 什麼是 Prompt

Prompt（提示詞）是你對 AI 下達的指令。好的 Prompt 能讓 AI 給你精準、有用的回答；模糊的 Prompt 只會得到模糊的結果。

```
壞 Prompt：              好 Prompt：
─────────────────       ───────────────────────
「設計一個網站」          「設計一個給 25-35 歲
                        上班族女性使用的
                        健身 app 登陸頁面，
                        風格清新自然，
                        主要 CTA 是免費試用」
```

## Prompt 的基本要素

一個完整的 Prompt 應該包含：

| 要素 | 說明 | 範例 |
|------|------|------|
| **角色** | AI 要扮演什麼身份 | 「你是一個專業的網頁設計師」 |
| **任務** | 具體要做什麼 | 「幫我設計一個登陸頁的結構」 |
| **背景** | 相關的上下文 | 「這是一個線上課程的促銷頁面」 |
| **目標** | 期望的結果 | 「目的是提高報名轉換率」 |
| **限制** | 特別的要求 | 「要在 500 字以內」 |
| **格式** | 輸出格式 | 「用表格呈現」 |

## Prompt 框架：ICIO

### I - Instruction（指令）

清楚說明你要 AI 做什麼：

```
❌「幫我想想」
✓「幫我列出 10 個」
✓「幫我撰寫一段」
✓「幫我檢查並修正」
```

### C - Context（背景）

提供足夠的上下文資訊：

```
沒有 Context：       有 Context：
─────────────────   ───────────────────────
「怎麼減肥」         「我是一個上班族，每天
                    坐在辦公室 8 小時，
                    只有中午休息 1 小時可以
                    運動，要怎麼減肥？」
```

### I - Input（輸入）

如果有任何需要處理的資料，提供給 AI：

```
「請根據以下產品資訊，幫我撰寫文案：

產品：[產品名稱]
功能：[功能描述]
目標客群：[描述]」

＋ 具體的產品資訊
```

### O - Output（輸出）

說明你期望的輸出格式：

```
「請用以下格式呈現：
1. 標題（8 字以內）
2. 副標題（20 字以內）
3. 三個重點（每點 15 字以內）
4. CTA 按鈕文字（10 字以內）」
```

## 不同場景的 Prompt 範例

### 場景 1：網站文案

```
角色：你是一個經驗豐富的網頁文案專家，擅長撰寫
      高轉換率的登陸頁文案。

任務：幫我撰寫一個線上課程登陸頁的 Hero 區文案。

背景：
- 課程名稱：Excel 從零到進階
- 目標客群：職場新鮮人、想提升效率的上班族
- 課程賣點：不用死記函數、案例式學習、30 天見效
- 價格：原價 2999，優惠價 1499

要求：
1. 主標題：痛點切入，引起共鳴
2. 副標題：說明課程核心價值
3. 3 個 Bullet Points：課程特色
4. CTA 按鈕文字
5. 語氣要親近，不要太商業化

格式：用 Markdown 格式呈現
```

### 場景 2：程式碼輔助

```
角色：你是一個專精前端開發的工程師。

任務：幫我寫一個 CSS 程式碼，讓我的網站 Hero 區
      有漸層背景。

需求：
1. 漸層從 #667eea 到 #764ba2（左上到右下）
2. 要完全覆蓋 Hero 區
3. 文字要白色且有陰影
4. 響應式：手機上 padding 要減少

程式碼要用現代 CSS（CSS3），避免過時語法。
```

### 場景 3：圖片生成

```
用 Midjourney 或 DALL-E：

主題：Modern minimalist home office setup
風格：Photography, clean aesthetic
光線：Natural window light
色彩：Warm wood tones, white, green plants
其他：MacBook on desk, coffee cup, not cluttered
比例：16:9
```

### 場景 4：網站架構規劃

```
角色：你是一個經驗豐富的 UX 設計師和資訊架構師。

任務：幫我規劃一個電商網站的頁面結構。

背景：
- 產品類型：有机保養品
- 目標客群：25-40 歲注重健康的女性
- 主要功能：產品銷售、會員系統、部落格

需求：
1. 列出所有必要的頁面
2. 說明每個頁面的主要目的
3. 建議頁面之間的導航結構
4. 指出哪些功能是「必要」vs「可選」

用表格呈現，格式清晰。
```

## 提升 Prompt 效果的技巧

### 技巧 1：分解複雜任務

```
一次完成（不好）：
「幫我設計整個網站，包括首頁、關於頁、服務頁，
每頁都要有文案和排版建議」

分步進行（推薦）：
步驟 1：「幫我規劃網站結構和頁面」
步驟 2：「針對首頁，幫我撰寫文案」
步驟 3：「針對首頁，給我排版建議」
...以此類推
```

### 技巧 2：給予範例

```
❌「用專業語氣寫」

✓「用以下範例的語氣寫：
「專為台灣人設計的課程，
不用出國也能學到原汁原味的美式口音」」
```

### 技巧 3：限制輸出範圍

```
❌「給我一些標題建議」

✓「給我 5 個標題建議，每個不超過 15 個字，
風格要有創意但不標題黨」
```

### 技巧 4：要求多重輸出

```
❌「給我一個標題」

✓「給我 3 個不同風格的標題：
1. 功能導向
2. 情感導向
3. 數據導向」
```

### 技巧 5：迭代優化

第一次 prompt 後，根據 AI 的回答再調整：

```
第一輪：「幫我寫一個網站標語」
AI 回覆後...

第二輪：「很好！但再俏皮一點，
        有沒有更年輕化的版本？」
```

## 常見 Prompt 錯誤

### 錯誤 1：太過籠統

```
❌「網站要好看」
✓「網站要北歐極簡風，白色為主，輔以木質元素，
   留白要多，給人溫暖舒適的感覺」
```

### 錯誤 2：一次要求太多

```
❌「幫我設計整個網站、寫所有文案、配色方案、
   字體選擇、圖片建議」
✓ 每個需求分開問
```

### 錯誤 3：忽略格式要求

```
❌「列點說明」→ AI 用段落呈現
✓「請用 Markdown 列表格式呈現，每點不超過 20 字」
```

### 錯誤 4：沒有說明用途

```
❌「幫我寫產品描述」→ AI 不知道用在哪裡
✓「幫我寫一個電商網站的產品卡描述，
   100 字以內，要放在手機螢幕上讓人快速掃讀」
```

## 進階 Prompt 技巧

### Chain of Thought（思考鏈）

要求 AI 逐步思考：

```
「在給我答案之前，先說明你思考這個問題的邏輯：
1. 我的需求是什麼？
2. 考量因素有哪些？
3. 為什麼你建議這個方案？」
```

### Few-Shot Learning（範例學習）

提供幾個例子讓 AI 理解你想要的風格：

```
「幫我寫產品標題，風格參考以下範例：
範例 1：減肥不求人，在家就能動
範例 2：懶人整理術，一招讓家裡變整齊
範例 3：30 天英文脱胎換骨

現在幫我為「線上瑜珈課程」寫 5 個標題」
```

### Role Playing（角色扮演）

讓 AI 扮演特定角色：

```
「你現在是一個：
- 頂尖的網頁設計師
- 精通 UX 心理學
- 說話直接，會給批評性建議
- 重視轉換率而非純美觀

我有一個登陸頁想要你給建議：[描述你的頁面]

請從設計師和行銷人的角度給我 feedback」
```

## 建立你的 Prompt 模板庫

建議建立一個文件，收集你常用的 Prompt：

```
# 常用 Prompt 模板

## 網站文案

### Hero 區文案
---
角色：你是一個...
任務：...
背景：...
要求：...
格式：...
---

### 產品描述
---
...
---
```

這樣下次需要類似任務時可以直接複製修改。

## 實作練習

### 練習 1：撰寫網站文案 Prompt

試著為以下情境撰寫完整的 Prompt：

情境：「你要為一個新開的咖啡店製作網站，需要首頁文案」

使用 ICIO 框架，完整描述角色、任務、背景、輸出格式。

### 練習 2：優化現有 Prompt

將以下模糊的 Prompt 改成詳細的 Prompt：

```
原始：「幫我寫文案」

情境：為一個線上英語課程寫推廣文案
```

### 練習 3：建立你自己的 Prompt

為你正在做的專案，設計一個完整的 Prompt，
存到你的模板庫中。
