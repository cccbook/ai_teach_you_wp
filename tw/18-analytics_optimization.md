# 18. 網站分析與持續優化

## 為什麼需要網站分析

建立網站只是第一步，之後需要持續觀察和優化：

> **沒有數據就沒有方向，不知道怎麼改進**

### 常見問題與數據解答

| 問題 | 答案來自 |
|------|----------|
| 多少人來過我的網站？ | 流量分析 |
| 從哪裡來的？ | 流量來源分析 |
| 他們看了哪些頁面？ | 頁面分析 |
| 為什麼離開了？ | 跳出率、退出頁面 |
| 他們做了我想讓他們做的事嗎？ | 轉換追蹤 |

## Google Analytics 4（GA4）設定

### 什麼是 Google Analytics

Google Analytics 是最流行的網站分析工具，可以追蹤：
- 訪客人數
- 流量來源
- 頁面瀏覽
- 使用者行為
- 轉換目標

### 免費申請

```
1. 前往 analytics.google.com
2. 點擊「開始測量」
3. 登入 Google 帳號
4. 填寫帳戶名稱（可以任意命名）
5. 填寫媒體資源名稱（通常是你的網站名）
6. 填寫商家資訊
7. 選擇數據分享設定
8. 取得「衡量 ID」（G-XXXXXXXXXX）
```

### 安裝追蹤碼

根據你的平台選擇方式：

**WordPress**：
```
使用外掛，例如：
- SiteKit by Google（推薦）
- GA Google Analytics
1. 安裝外掛
2. 填入你的 G-XXXXXX ID
3. 儲存設定
```

**Wix、Squarespace 等**：
```
通常在後台有內建整合：
1. 進入網站設定
2. 找到「分析」或「第三方工具」
3. 貼上 GA ID
```

**自架網站（HTML）**：
```html
在 <head> 標籤中加入：
<!-- Google tag (gtag.js) -->
<script async src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXX"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'G-XXXXXX');
</script>
```

## 認識 Google Analytics 4 介面

### 主要報告

| 報告 | 你能看到 |
|------|----------|
| 即時 | 目前在網站上的人 |
| 生命週期報告 → 總覽 | 整體流量趨勢 |
| 生命週期報告 → 取得 | 訪客從哪裡來 |
| 生命週期報告 → 參與 | 哪些頁面最受欢迎 |
| 生命週期報告 → 營利 | 電子商務數據 |
| 生命週期報告 → 保留 | 回訪者比例 |
| 探索 | 自訂分析 |

### 關鍵指標

| 指標 | 說明 | 什麼是「好」 |
|------|------|-------------|
| 使用者 | 總訪客人數 | 越多越好（但要看品質） |
| 工作階段 | 訪問次數 | 越多越好 |
| 跳出率 | 只看一頁就離開的比例 | 越低越好（<50% 佳） |
| 平均工作階段時間 | 平均停留時間 | 越長越好 |
| 網頁瀏覽量 | 總共看了多少頁 | 越多越好 |
| 轉換 | 完成目標的次數 | 越多越好 |

## Google Search Console（GSC）

### 與 GA4 的差異

| 工具 | 用途 |
|------|------|
| Google Analytics | 了解使用者行為 |
| Search Console | 了解搜尋表現 |

Search Console 告訴你：
- 你的網站在 Google 搜尋中的表現
- 哪些關鍵字帶來流量
- 網站有沒有技術問題
- 網頁是否被正確索引

### 基本設定

```
1. 前往 search.google.com/search-console
2. 點擊「立即開始」
3. 輸入你的網址
4. 選擇驗證方法
5. 完成驗證
```

### 驗證方法

| 方法 | 優點 | 缺點 |
|------|------|------|
| HTML 檔案上傳 | 簡單快速 | 需要能上傳檔案 |
| HTML 標籤 | 最簡單 | - |
| 網域名稱供應商 | 最完整 | 需要變更 DNS |
| Google tag | GA4 已設定的話最簡單 | - |

### 查看關鍵報告

**成效報告**：
- 你的網站出現了哪些搜尋？
- 點擊率是多少？
- 平均排名為何？

**索引覆蓋範圍**：
- 有多少頁面被 Google 索引？
- 有沒有錯誤？

**Core Web Vitals**：
- 網站速度體驗評分
- 行動裝置可用性

## 使用 AI 分析數據

### 常見的數據分析 Prompt

```
分析網站數據 Prompt：

「我是一個網站站長，想了解如何優化我的網站。
以下是上個月的數據：

- 總訪客：5,000
- 跳出率：65%
- 平均停留時間：1 分 30 秒
- 主要流量來源：Google 搜尋 (60%)、Facebook (20%)、直接輸入 (20%)
- 最受歡迎頁面：首頁、關於頁、作品集頁
- 跳出率最高的頁面：聯絡頁 (80%)

請分析：
1. 這些數據顯示什麼問題？
2. 我應該優先優化什麼？
3. 有什麼具體建議？」
```

### 分析流量下降的原因

```
Prompt：

「我的網站流量最近下降了 30%，
可能的原因有哪些？
我應該檢查什麼？」

「以下是流量數據：
[貼上數據]

請幫我分析可能的原因。」
```

### 建議網站優化方向

```
Prompt：

「基於以下用戶行為數據，
建議我應該優先優化哪些頁面和功能：

[貼上頁面瀏覽量、跳出率、轉換率等數據]」
```

## 持續優化策略

### 定期檢查項目

建議每週或每月檢查：

| 時間 | 檢查項目 |
|------|----------|
| 每週 | 流量趨勢、有沒有異常 |
| 每月 | 完整數據報告、與上月比較 |
| 每季 | 內容表現、競爭對手分析 |
| 每年 | 整體策略檢視 |

### 常見優化方向

```
1. 內容優化
   - 更新過時內容
   - 擴展表現好的內容
   - 移除表現差的內容

2. 技術優化
   - 提升網站速度
   - 修復錯誤頁面
   - 改善手機版體驗

3. 轉換優化
   - 測試不同的 CTA
   - 簡化表單
   - 減少購物流程步驟

4. SEO 優化
   - 針對新關鍵字優化
   - 建立內部連結
   - 爭取外部連結
```

## 網站速度優化

### 為什麼速度很重要

| 載入時間 | 跳出率影響 |
|---------|-----------|
| 1 秒 | 32% 跳出率 |
| 3 秒 | 50% 跳出率 |
| 5 秒 | 90% 跳出率 |

### 速度檢測工具

| 工具 | 網址 | 特點 |
|------|------|------|
| PageSpeed Insights | pagespeed.web.dev | Google 官方，包含 Core Web Vitals |
| GTmetrix | gtmetrix.com | 詳細分析報告 |
| WebPageTest | webpagetest.org | 最詳細但複雜 |

### 速度優化建議

```
AI 給的優化建議 Prompt：

「我的網站在 PageSpeed Insights 的分數是：
- 效能：45 分
- 可惜化內容最大區塊：3.2MB
- 哪些項目需要優化：[問題列表]

請給我具體的優化建議，
包括：哪些要優先處理，
以及如何處理？」
```

## A/B 測試基礎

### 什麼是 A/B 測試

同時測試兩個版本，看哪個效果更好。

```
訪客
  ↓
隨機分組
  ↓
┌──────┐    ┌──────┐
│ A 版 │    │ B 版 │
│(控制組)│    │(實驗組)│
└──────┘    └──────┘
  ↓           ↓
收集數據 → 分析哪個更好
```

### 常見測試項目

| 測試項目 | A 版本 | B 版本 |
|---------|--------|--------|
| CTA 按鈕顏色 | 藍色 | 紅色 |
| CTA 按鈕文字 | 「了解更多」 | 「立即註冊」 |
| 主標題 | 功能導向 | 利益導向 |
| 圖片 | 產品圖 | 情境圖 |
| 表單欄位數 | 5 個 | 3 個 |

### A/B 測試工具

| 工具 | 說明 |
|------|------|
| Google Optimize（已停用） | 免費但已停止服務 |
| VWO | 功能完整，有免費版 |
| Optimizely | 企業級方案 |
| Nelio A/B Testing | WordPress 外掛 |

## 常見問題與解決

### Q：網站流量很低怎麼辦？

```
可能原因：
1. 還沒被 Google 索引 → 提交到 Search Console
2. 關鍵字選擇錯誤 → 研究目標關鍵字
3. 內容不夠好 → 提升內容品質
4. 缺乏推廣 → 增加外部連結、社群分享
5. 技術問題 → 檢查 robots.txt、sitemap

解決順序：
1. 先確認技術上沒問題
2. 優化 SEO
3. 提升內容品質
4. 加強推廣
```

### Q：跳出率很高怎麼辦？

```
跳出率高代表訪客很快就離開，可能原因：

1. 載入速度慢 → 優化速度
2. 內容不符預期 → 改善標題和預覽
3. 網站難以使用 → 改善 UX
4. 行動呼籲不明顯 → 加強 CTA
5. 手機版體驗差 → 確保響應式設計

檢查方法：
- 看哪些頁面跳出率最高
- 用熱點工具（如 Hotjar）看用戶行為
- 收集用戶回饋
```

### Q：網站被駭怎麼辦？

```
預防：
1. 保持外掛/主題更新
2. 使用強密碼 + 兩步驟驗證
3. 安裝資安外掛
4. 定期備份
5. 使用 SSL（HTTPS）

處理：
1. 保持冷靜
2. 聯繫主機商請求協助
3. 從備份還原
4. 變更所有密碼
5. 檢查並修復漏洞
6. 通知 Google（如果需要的話）
```
