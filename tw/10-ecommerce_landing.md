# 10. 設計電商登陸頁面

## 什麼是登陸頁面 (Landing Page)

登陸頁面是一個「專門為某個目標設計的單一頁面」，不同於一般網站有多個頁面導航，登陸頁面的目標只有一個：

> **說服訪客完成一個特定的行動**

### 登陸頁 vs 一般網站

| 項目 | 登陸頁 | 一般網站 |
|------|--------|---------|
| 目標 | 單一轉換目標 | 多種目標 |
| 連結 | 少（保持專注） | 多（方便瀏覽） |
| 導航 | 極簡或無 | 完整 |
| 適合場景 | 促銷、活動、產品發布 | 品牌形象、長期經營 |

### 常見登陸頁類型

1. **產品發布**：介紹新產品
2. **限時促銷**：節日、周年慶活動
3. **免費體驗**：試用、样品索取
4. **課程報名**：線上課程、工作坊
5. **活動報名**：講座、event

## AI 幫你設計登陸頁

### 使用 Durable 建立

在 Durable 中選擇「Landing Page」類型：

```
「我要建立一個 [產品名稱] 的登陸頁面，
這是一個 [產品類型]，
主要功能是 [產品功能]，
目標是 [促銷報名/銷售/搜集名單]，
限時優惠是 [優惠內容]（如果有的話）」
```

### 使用 Wix ADI 建立

選擇「Landing Page」模板，回答 AI 的問題：

1. 這個頁面的目的是什麼？
2. 你想推廣什麼產品或服務？
3. 有沒有特別的優惠或 CTA？

## AI 文案生成 Prompt

### Hero 區文案

```
角色：你是轉化率優化專家

任務：幫我撰寫登陸頁 Hero 區的文案

產品：[產品名稱]
產品類型：[產品類型]
主要功效：[產品能解決什麼問題]
目標客群：[誰會買這個產品]

需求：
1. 主標題（8 字以內，強而有力）
2. 副標題（一句話說明最大賣點）
3. CTA 按鈕文字（10 字以內，行動導向）

範例風格：
主標題：「減肥不再是痛苦的代名詞」
副標題：「懶人也能輕鬆瘦的革命性方法」
CTA：「免費試用 14 天」
```

### 痛點共鳴文案

```
角色：你是一個了解目標客戶心理的文案專家

任務：幫我撰寫痛點共鳴區的文案

產品：[產品名稱]
目標客群的痛點：[客戶通常遇到的問題]

需求：
1. 列出 3-4 個痛點
2. 每個痛點用一句話描述，要有畫面感
3. 語氣像是「我懂你」而不是「你在錯」
4. 格式： emoji + 短句 + 痛感描述

格式範例：
😫 減肥失敗無數次
   每次下定決心，最後都半途而廢...

🍕 美食當前無法抗拒
   減肥計劃永遠從明天開始...
```

### 產品特色文案

```
角色：你是一個專業的產品文案專家

任務：幫我撰寫產品特色區的文案

產品：[產品名稱]
產品特色：[3-4 個主要特色]

需求：
1. 每個特色用標題 + 說明的方式呈現
2. 標題要具體、動感強
3. 說明要呼應痛點，說明能帶來什麼改變
4. 避免「高品質」「最棒」這類空泛形容詞

格式範例：
🚀 30 秒快速上手
   不需要說明書，開箱就能用

💰 省下 50% 的時間
   原本要 2 小時的工作，現在 1 小時搞定
```

### 客戶見證文案

```
角色：你是一個擅長撰寫客戶心聲的文案專家

任務：幫我撰寫 3 個客戶見證

產品：[產品名稱]
目標客群：[典型的購買者]
常見的使用情境：[客戶通常怎麼用這個產品]

需求：
1. 3 個不同背景的見證者
2. 每個見證包含：
   - 使用前的困擾
   - 使用後的改變（最好有數據）
   - 一句總結
3. 見證要像真實的 user feedback
4. 可以加入職業、年齡等背景讓見證更有血有肉

格式範例：
「我是 [背景]，之前 [困擾]，
用了 [產品] 以後，[具體改變]，
[總結心得]。」
— [名字]，[職業]
```

### FAQ 文案

```
角色：你是一個客服專家

任務：幫我撰寫常見 FAQ

產品：[產品名稱]
目標客群：[購買者可能的疑問]

需求：
1. 列出 4-6 個最常見的問題
2. 問題要具體，不是「這個產品好嗎？」
3. 回答要詳細、消除疑慮
4. 回答最後可以順便再推銷一下產品

常見 FAQ 方向：
- 適合我嗎？
- 多少錢？有沒有優惠？
- 運費與配送？
- 退換貨政策？
- 如何使用？
- 跟其他產品有什麼不同？
```

## 登陸頁視覺設計技巧

### 主視覺設計原則

1. **產品要清楚**：一眼就能看出是什麼產品
2. **情境圖加分**：產品在生活中的使用情境
3. **前後對比**：Before/After 視覺衝擊力強

### CTA 按鈕設計

CTA（Call to Action）是登陸頁最重要的元素：

**顏色要突出**：
- 與頁面主色形成對比
- 紅、橙、綠是常見的 CTA 顏色
- 避免與背景融為一體

**文案要具體**：
- ❌ 「提交」「送出」「按這裡」
- ✅ 「免費試用」「立即領取」「馬上報名」

**位置要多處出現**：
- Hero 區
- 特色介紹後
- 見證區後
- FAQ 區後
- 頁尾

### 消除疑慮的元素

讓顧客買得安心：

1. **安全支付標章**：信用卡公司 logo、SSL 標誌
2. **退換貨保證**：30 天退貨、無條件退款
3. **免運費標示**：全站免運、X 金額免運
4. **限量/限時**：庫存緊張、優惠倒數
5. **用户數據**：已有 X 萬人購買

## 登陸頁檢查清單

### 轉換優化檢查

- [ ] 頁面載入時間 < 3 秒
- [ ] 行動呼籲按鈕清晰可見
- [ ] 手機版 CTA 按鈕夠大（至少 44px）
- [ ] 价值主張在 5 秒內清楚傳達
- [ ] 有至少 2-3 個見證
- [ ] FAQ 回答了常見疑慮
- [ ] 最後有緊急感元素（倒數、庫存）

### 技術檢查

- [ ] 所有連結正常
- [ ] 表單能正常提交
- [ ] GA/追蹤代碼正確安裝
- [ ] 手機版顯示正常
- [ ] 圖片有替代文字 (Alt Text)
- [ ] 有 SSL 憑證

### 內容檢查

- [ ] 標題吸睛
- [ ] 產品功能說明清楚
- [ ] 價格透明
- [ ] 聯絡方式方便找到
- [ ] 沒有錯字和文法錯誤

## A/B 測試概念

當你不確定哪個版本更好時，可以做 A/B 測試：

### 測試項目建議

| 測試項目 | A 版本 | B 版本 |
|---------|--------|--------|
| CTA 按鈕顏色 | 紅色 | 綠色 |
| 主標題 | 功能導向 | 利益導向 |
| 圖片 | 產品照 | 情境照 |
| 價格呈現 | 原價划掉 | 直接顯示優惠價 |

### 測試時間

每個版本至少運行 2-4 週，且每個版本至少 1000 次訪問。
